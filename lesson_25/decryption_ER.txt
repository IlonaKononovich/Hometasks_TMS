Задача для самостоятельной работы:
Вы разрабатываете систему управления онлайн-курсами. В системе участвуют преподаватели, студенты, курсы, модули, уроки, домашние задания и оценки.
1. Выделить все сущности и их атрибуты (указать существующие и придумать по 1 или более дополнительному по необходимости)
2. Найдите связи между сущностями. Укажите типы связей: один ко многим (1:M), один к одному (1:1), многие ко многим (M:N)
3. Постройте ER-диаграмму


ПОЯСНЕНИЯ:Связи между таблицами следующие:
1. 
- Преподаватель может вести несколько курсов.

Связь между teachers и courses  - многие ко многим, реализуется через дополнительную таблицу teachers_courses. Один учитель может вести много курсов. Один курс может преподаваться многими учителями.

Таблица teachers и teachers_courses - связь один ко многим. 
Один учитель (из teachers) может иметь много записей в teachers_courses (например, для разных курсов).Каждая запись в teacher_courses относится только к одному учителю.

Таблица courses и teacher_courses - связь один ко многим. 
Один курс (из courses) может иметь много записей в teacher_courses (например, для разных учителей).Каждая запись в teacher_courses относится только к одному курсу.

2. 
- Курс состоит из нескольких модулей

Таблица courses связана с таблицей modules - один ко многим
Один курс (из таблицы courses) может содержать несколько модулейКаждый модуль (из таблицы modules) принадлежит только одному курсу
3.- Модуль содержит несколько уроков
Связь между таблицами modules и lessons - один ко многим. Один модуль содержит много уроков, а каждый урок принадлежит только одному модулю.

4. 
- Урок может содержать домашнее задание

Связь между lessons и homework  - многие ко многим, реализуется через дополнительную таблицу lessons_ homework. 
Один урок может включать много заданий (через записи в lesson_homework).Одно задание может быть привязано к нескольким урокам (например, общее задание для тем "Функции" и "Классы").

Таблица lessons и  lessons_ homework - связь один ко многим. 
Один урок (из lessons) может иметь много записей в lessons_ homework.Каждая запись в lessons_ homework относится только к одному уроку.

Таблица homework и lessons_ homework - связь один ко многим. 
Одно домашнее задание (из homework) может иметь много записей в lessons_ homework . Каждая запись в lessons_ homework относится только к одному домашнему заданию.

5.
- Студент может записаться на несколько курсов.

Связь между students и courses - многие ко многим, реализуется через дополнительную таблицу students_courses. Один студент может записаться на много курсов. Один курс может содержать много студентов.

Таблица students и students_courses - связь один ко многим.
Один студент (из students) может иметь много записей в students_courses (например, для разных курсов).Каждая запись в students_courses относится только к одному студенту.


Таблица courses и students_courses - связь один ко многим.
Один курс (из courses) может иметь много записей в students_courses (например, для разных студентов).Каждая запись в students_courses относится только к одному курсу.

6. и 7. 
- Студент выполняет домашние задания.
- Преподаватель ставит оценку за каждое выполненное задание.

Связь между students и homework - многие ко многим, реализуется через таблицу student_homework. Один студент может выполнить много домашних заданий. Одно домашнее задание может быть выполнено многими студентами.
Таблица students и student_homework - связь один ко многим.
Один студент (из students) может иметь много записей в student_homework (например, для разных заданий). Каждая запись в student_homework относится только к одному студенту.

Таблица homework и student_homework - связь один ко многим.
Одно домашнее задание (из homework) может иметь много записей в student_homework (например, выполнено разными студентами). Каждая запись в student_homework относится только к одному домашнему заданию.

Таблица teachers и student_homework - связь один ко многим.
Один учитель (из teachers) может проверить много заданий в student_homework. Каждая запись в student_homework относится только к одному проверяющему учителю (может быть NULL, если задание еще не проверено).

Таблица grades и student_homework - связь один к одному.
Каждая оценка (из grades) связана ровно с одной записью в student_homework (через grade_id). Каждая запись в student_homework может ссылаться только на одну оценку (может быть NULL, если задание еще не оценено).


8.
- У каждого студента есть профиль , содержащий имя, email, дату регистрации.

Связь между students и courses - регистрация студентов на курсах многие ко многим, реализуется через таблицу student_course_registrations. Один студент может зарегистрироваться на много курсов. Один курс может содержать много зарегистрированных студентов.
Таблица students и student_course_registrations - связь один ко многим.
Один студент (из students) может иметь много записей в student_course_registrations (например, регистрации на разные курсы). Каждая запись в student_course_registrations относится только к одному студенту.

Таблица courses и student_course_registrations - связь один ко многим.
Один курс (из courses) может иметь много записей в student_course_registrations (например, регистрации разных студентов). Каждая запись в student_course_registrations относится только к одному курсу.


9. 
- Каждый курс имеет категорию (например, "Программирование", "Маркетинг").

Реализовано через добавление артибута course_category в таблице cources

10.
- Курс может иметь отзывы от студентов.

Связь между students и courses - отзывы студентов о курсах многие ко многим, реализуется через таблицу course_reviews. Один студент может оставить много отзывов (на разные курсы). Один курс может иметь много отзывов (от разных студентов).

Таблица students и course_reviews - связь один ко многим.
Один студент (из students) может иметь много записей в course_reviews (например, отзывы на разные курсы). Каждая запись в course_reviews относится только к одному студенту.

Таблица courses и course_reviews - связь один ко многим.
Один курс (из courses) может иметь много записей в course_reviews (например, отзывы разных студентов). Каждая запись в course_reviews относится только к одному курсу.
